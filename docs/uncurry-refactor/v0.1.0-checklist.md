# v0.1.0 Release Checklist

This checklist tracks the steps required to release version `0.1.0` of `fp-library`, which involves removing the legacy v1 API and promoting the v2 API to the crate root.

## Phase 1: Code Cleanup (Removal of v1)

- [x] **Extract Brand Definitions**:
    - [x] Update `fp-library/src/brands.rs` to define the brand structs directly instead of re-exporting them from `crate::types`.
    - [x] Brands to define:
        - `ArcFnBrand`
        - `EndofunctionBrand`
        - `EndomorphismBrand`
        - `IdentityBrand`
        - `LazyBrand`
        - `OnceCellBrand`
        - `OnceLockBrand`
        - `OptionBrand`
        - `PairBrand`, `PairWithFirstBrand`, `PairWithSecondBrand`
        - `RcFnBrand`
        - `ResultBrand`, `ResultWithErrBrand`, `ResultWithOkBrand`
        - `VecBrand`
- [x] **Remove Feature Flags**:
    - [x] Remove `v1` and `v2` features from `fp-library/Cargo.toml`.
    - [x] Remove `#[cfg(feature = ...)]` guards and `#[deprecated]` attributes from `fp-library/src/lib.rs`.
- [x] **Delete Legacy Modules**:
    - [x] Delete `fp-library/src/classes/` directory.
    - [x] Delete `fp-library/src/types/` directory.
    - [x] Delete `fp-library/src/functions.rs` file.

## Phase 2: Promotion of v2

- [x] **Move v2 to Root**:
    - [x] Move contents of `fp-library/src/v2/classes/` to `fp-library/src/classes/`.
    - [x] Move contents of `fp-library/src/v2/types/` to `fp-library/src/types/`.
    - [x] Move `fp-library/src/v2/functions.rs` to `fp-library/src/functions.rs`.
    - [x] Delete `fp-library/src/v2/` directory.
- [x] **Update `lib.rs`**:
    - [x] Ensure `classes`, `types`, and `functions` are declared as public modules (`pub mod ...`).
    - [x] Remove `pub mod v2;`.

## Phase 3: Internal Refactoring

- [x] **Fix Internal Imports**:
    - [x] Search and replace `crate::v2::` with `crate::` across the entire codebase.
    - [x] Fix any broken relative imports (`super::`).
- [x] **Verify String Implementation**:
    - [x] Verify `fp-library/src/types/string.rs` contains the `Kind1L0T` implementation and no feature flags. (Note: This file will be the one moved from `v2/types/string.rs`).

## Phase 4: Verification and Documentation

- [x] **Run Tests**:
    - [x] Run `cargo test` to ensure all tests pass.
    - [x] Verify documentation tests compile.
- [x] **Update Documentation**:
    - [x] Update `README.md` to remove "v2" references and migration guides.
    - [x] Update crate-level documentation in `fp-library/src/lib.rs`.

## Phase 5: Release

- [x] **Bump Version**:
    - [x] Update `fp-library/Cargo.toml` version to `0.1.0`.
- [ ] **Publish**:
    - [ ] Commit changes.
    - [ ] Tag `v0.1.0`.
    - [ ] Run `cargo publish`.
