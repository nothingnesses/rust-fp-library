# Stage 1 classification: aggregated findings

**Status:** pending
**Last updated:** _not yet started_

## Purpose

Aggregated synthesis of every per-codebase Stage 1 file in this directory.
Produced only after every entry under "Stage 1: per-codebase
classification" in [\_status.md](_status.md) is ticked. The purpose is to
turn 13 individual findings into a decision artefact: which codebases are
variants of options 1-5 in [../port-plan.md](../port-plan.md) section 4.1,
which are genuinely novel, and which (if any) warrant Stage 2 deep dives.

This file is intentionally empty until Stage 1 completes. The contract
for filling it in is prompt 3 in [prompts.md](prompts.md).

## Required structure (to be populated later)

An agent filling in this document must produce the following sections.
Each heading below is a placeholder; replace the placeholder content with
real findings.

### Intro

_(One paragraph: what Stage 1 set out to answer and what this synthesis
delivers.)_

### Classification table

_(One row per codebase. Columns: codebase name, core substrate,
classification against port-plan section 4.1 options 1-5, one-line
novelty verdict.)_

### Novel encodings

_(One subsection per codebase flagged genuinely novel in Stage 1. Brief
sketch of what makes it novel and why it deserves a Stage 2 deep dive.)_

### Variants of known options

_(One paragraph per option family summarising which codebases cluster
there and whether any contributed new insight within that family.)_

### Recommendations for Stage 2

_(Which, if any, Stage 2 deep dives are worth scheduling. For each, name
the specific question it should answer and which codebase file it should
extend.)_

### Relevance to port-plan

_(Which sections of ../port-plan.md would change based on Stage 1
findings, or "no changes recommended" explicitly if that is the
conclusion.)_
